# 🍔 burgerHouse - Bestell-App

Eine moderne und reaktionsschnelle (responsive) Webanwendung für ein Burger-Restaurant, über die Kunden das Menü einsehen und Bestellungen aufgeben können. Das Projekt trennt Logik, Styling und Struktur strikt nach modernen Best Practices.

## 🚀 Features
* **Interaktives Menü:** Übersichtliche Präsentation von Burgern, Beilagen und Getränken.
* **Warenkorb & Bestellung:** Dynamisches Hinzufügen von Artikeln und automatische Preisberechnung.
* **Responsive Design:** Optimierte Darstellung sowohl auf Smartphones als auch auf Desktop-Bildschirmen.

## 🛠️ Verwendete Technologien
Das Projekt wurde vollständig mit den nativen Standards des modernen Web-Frontends umgesetzt:
* **HTML5:** Strukturierung der Inhalte und Formulare (`index.html`, `html/`).
* **CSS3:** Modernes Layout-Design mit Flexbox/Grid und Animationen (`style.css`, `styles/`).
* **JavaScript (ES6+):** Dynamische DOM-Manipulation, Event-Handling und Bestelllogik (`script.js`, `scripts/`).

## 📂 Projektstruktur
```text
├── assets/           # Bildmaterial (Burger-Fotos, Icons, Logos)
├── html/             # Optionale Unterseiten oder HTML-Module
├── scripts/          # Strukturierte JavaScript-Module und Validierungen
├── styles/           # Aufgeteilte CSS-Stylesheets für bessere Übersicht
├── index.html        # Die Hauptseite der Anwendung
├── script.js         # Zentrale JavaScript-Logik
├── style.css         # Haupt-Stylesheet
└── .gitignore        # Ausschluss von Systemdateien für Git
```

## 💻 Lokale Einrichtung & Start

Da dieses Projekt auf nativem Frontend-Code basiert, ist keine komplexe Server-Installation oder Node.js-Laufzeitumgebung erforderlich.

### Voraussetzungen
Sie benötigen lediglich einen modernen Webbrowser (z. B. Google Chrome, Mozilla Firefox, Microsoft Edge).

### Starten der App
1. **Repository klonen:**
   ```bash
   git clone https://github.com/honkytong889/burgerHouse

   ```
2. **In das Projektverzeichnis wechseln:**
   ```bash
   cd burgerHouse
   ```
3. **Im Browser öffnen:**
   * Machen Sie einfach einen Doppelklick auf die Datei `index.html` in Ihrem Dateimanager.
   * *Alternativ in VS Code:* Nutzen Sie die Erweiterung **Live Server**, um die Anwendung lokal über einen lokalen Entwicklungs-Server zu starten.
